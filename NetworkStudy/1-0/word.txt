■ ネットワーク
LAN
 Local Area Network
 限定したエリアで用いるネットワーク
有線LAN
 ケーブルを使ったネットワーク、セキュリティが高い
無線LAN
 電波で接続したネットワーク、ケーブルが不要
WAN
 Wide Area Network
 LAN同士を繋いで、広範囲のネットワークを形成したもの
インターネット
 複数のコンピュータネットワークを通信プロトコルで相互接続したネットワークの総称

■ ネットワーク構成
ノード
 ネットワークに繋がっている1つ1つの機器を指す
リンク
 ノードとノードを繋げているもの（ケーブルなど）
トポロジ
 コンピュータネットワークの接続形態のこと
バス型トポロジ
 1本のケーブル(同軸ケーブル)に複数のノードを接続しているトポロジのこと
 ケーブルの両端にターミネータ(抵抗器)を取り付けて、終端での信号の反射、乱れを防ぐ
スター型トポロジ
 1つの集線装置(ハブやスイッチ)に複数のノードを接続しているトポロジのこと
 今日の主流
メッシュ型トポロジ
 多くのノードが相互接続する形態、WAN、インターネットで使われる
 一つの接続が切れても、迂回した接続があれば動作可能 
帯域
 単位時間に送信できるデータ転送量（最大量）
 「帯域（幅）が広い」＝「一度に多くのデータが転送できる」
bps
 bits per second
 1秒あたりに転送できるデータ数の単位
輻輳（ふくそう）
 通信回線やネットワークに想定を超える接続要求や伝送要求が行われ、通信・通話ができなくなる状態。パンク
コリジョン
 同じ伝送路を共有する複数の装置が同時に信号を発信して混信する現象
 衝突、激突、不一致
ドメイン
 インターネット上に存在するコンピューターやネットワークを識別するための名前、住所
ポート
 トランスポート層のプロトコルで用いられる、0から65535までの番号が付与された論理的な情報の送受信口

■ モデル
プロトコル
 コンピューター同士が通信を行うための規格（ルール）、通信規約
OSI参照モデル
 コンピューターネットワークに求められる機能（プロトコル）を7階層の構造に分割し定義したもの
ヘッダ
 そのデータや文書自体についての情報（制御情報）を記述した部分
 （データの順番や再送機能など）
カプセル化
 各層でヘッダを付加してデータグラムにすること
非カプセル化
 届いたデータを1層から順にヘッダをはがしていき、7層で生のデータとなりコンピュータに到着するプロセス
PDU
 コンピュータ間の通信において使用されるデータの単位のこと。
 データリンク層＝フレーム、ネットワーク層＝パケット、トランスポート層＝セグメント
MTU
 Maximum Transmission Unit
 ネットワーク上において一度の通信で転送可能なデータのサイズ制限 
フレーム
 データリンク層（イーサネット）において通信されるデータ単位、ヘッダ+パケット+トレーラからなる
パケット
 ネットワーク層（IP）において通信されるデータの単位、「ヘッダ+セグメント」「ヘッダ+データグラム」
セグメント
 トランスポート層（TCP）において通信されるデータの単位、TCPヘッダが頭につけられたもの
TCP/IPモデル
 Transmission Control Protocol / Internet Protocol
 インターネット通信イントラネット通信において最も利用されている通信プロトコル
 アプリケーション層、トランスポート層、インターネット層、ネットワークインターフェース層からなる

■ 物理層
 ビットや電気信号を正しく送受信する層
 コンピュータとケーブルを接続し、コンピュータが扱う2進数のデータとケーブルが扱う電気信号を相互に変換する層
ビット
 コンピュータが処理するために用いる0と1だけで構成されたデータ（2進数）の桁数
 1ビット＝「0」「1」、2ビット＝「00」「01」「10」「11」
ツイストペアケーブル
 2本の電線をより合わせたケーブルのこと、「より対線」
UTP
 Unshielded Twisted Pair
 8本の銅線を2本ずつより合わせた4対の線をさらにより、外側をビニール飛膜で覆ったケーブル
 安価で扱いやすい、外部からの影響を受けやすい
STP
 shielded Twisted Pair
 2本ずつより合わせた線をシールドで覆い、その外側をさらに金属箔などでシールド処理したケーブル
 ノイズ影響を受けにくい、コストが高いから特殊環境以外ではあまり使用されない
カテゴリー
 ツイストペアケーブルの品質
 数字が大きい→高品質、価格高、1秒あたりに送信できるデータ量多
コネクタ
 コンピュータと周辺機器、電子機器などを接続し、電気的に連絡可能とする接続器
ストレートケーブル
 8本の銅線を全く同じ並びでコネクタに接続しているケーブル
クロスケーブル
 8本の銅線をクロスさせて接続しているケーブル
 1,2番コネクタを3,6番コネクタに接続されている
 同じグループに属するノード同士を接続する際に使用
光ファイバケーブル
 電気信号ではなく、光によって通信するケーブル
 中心部のコアとその周りのクラッドをさらに皮膜で覆った構成
 回線速度が速く安定している、取扱が難しい
ハブ
 半二重通信を行う
 信号を整形・増幅したり、ケーブルを集約したりするネットワークデバイス
半二重通信
 一度に「送信」「受信」のどちらか片方しか行えない通信方法
 通信相手との間に1本経路がつながっていて、「送信」「受信」を交互に行う

■ データリンク層
 隣接する機器と通信する層
イーサネット
 代表的な優先LANの一種、
ファストイーサネット
 100Mbpsの速度でトラフィックを伝えるイーサネット規格の総称
 100BASE-X
ギガビットイーサネット
 1000Mbps
イーサネットヘッダ
 「Ethernet Ⅱ」「IEEE802.3」
 イーサネットフレームにあるデータ部分以外の通信情報を格納する場所
トレーラ
 エラー検出情報
CSMA/CD
 Carrier Sense Multiple Access with Collision Detection
 初期イーサネットLANで使用されていたアクセス制御方式
MACアドレス
 ネットワーク機器やネットワークアダプタに付いている固有の識別番号
 0～9、A～Fの16進数
 例 A0：B2：D5：(←ベンダーID)7F：（←機種ID）81：B3（←シリアルID）
スイッチ
 複数のコンピュータやLAN接続を行うネットワーク機器
 ポート（接続口）で受け取ったデータを、宛先のポートに送信する役割
MACアドレステーブル
 スイッチのポートとスイッチに接続されている、パソコン・ネットワーク機器のMACアドレス対応表
フラッディング
 ネットワーク機器の処理能力を超えるデータ量が発生し、ネットワークシステムの動作に支障が出ること
 「洪水・氾濫」
全二重通信
 「送信」「受信」を同時に行うことができる通信方法
 通信相手との間に、送信用と受信用の２本の経路がつながっている
VLAN
 Virtual LAN
 1つのLANの中で、論理的に仮想グループを設定することで、物理的な接続とは別のLANであるかのように運用できる技術
タグVLAN
 データに、そのデータが所属するネットワーク情報を付与し、制御する側はその情報を見て所属するネットワークを判断するやり方
ポートベースVLAN
 レイヤー2スイッチのポートに基づいてVLANの割り当てを行う方法
アクセスポート
 1つのVLANだけに所属するポート
トランクポート
 複数のVLANに所属するポート
DTP
 Dynamic Trunking Protocol
 スイッチポートをアクセスポートまたはトランクポートにするのかを動的に決定するシスコ社独自のプロトコル
デフォルトVLAN
 CatalystスイッチのVLAN1
 初期状態で設定されている
ネイティブVLAN
 トランクポートを流れるフレームにVLANタグが付与されない唯一のVLAN
ブロードキャストストーム
 ネットワークにおいて一斉送信用の特殊データや信号が転送され続ける現象
 2台以上のネットワークでループが形成されることで発生し、最終的にネットワークの全体機能が停止する
スパニングツリープロトコル
 円環上（ループ状）に形成されたレイヤー2ネットワークにおいてデータトラフィックが永続的に流れ続けることを防止するプロトコル
リンクアグリゲーション
 複数の物理リンクを束ねて1つの論理リンクとして扱うこと
 通信時に複数の物理リンクを同時に使用できる
 例 2ポートのGigabitEthernetポートをリンクアグリケーション→ 1Gbps × 2 = 2Gbps
EtherChannel
 複数の物理リンクを束ねて1つの論理リンクとして扱うこと（リンクアグリケーション）
 Ciscoでの呼称

■ ネットワーク層
 通信相手との最適な経路を判断する層
IP
 Internet Protocol
 ネットワーキングのためのプロトコル
IPヘッダ
 バージョン(4bit)、ヘッダ長(4bit)、サービスタイプ(8bit)、パケット長(16bit)、識別子(16bit)、
 フラグ(3bit)、フラグメントオフセット(13bit)、生存回数(8bit)、プロトコル(8bit)、チェックサム(16bit)、
 送信元IPアドレス(32bit)、宛先IPアドレス(32bit)、オプション(可変長)、パディング(可変長)
ICMP
 Internet Control Messeage Protocol
 IP通信の通信エラーに関する内容を通知する機能を持つ
ICMPメッセージ
 問い合わせやエラーに関するメッセージをやり取りすること
 IP通信の判定、フロー制御、Ping要求・応答、時間経過、Redirect機能、アドレスマスク要求・応答
IPアドレス
 ネットワークに接続されている機器に割り当てられた、ネットワークの構成要素を識別する番号
 ネットワーク部 + ホスト部
オクテット
 8ビット（2進数で8桁の数を表すことができる情報量）のこと
ネットワーク部
 そのIPアドレスが属しているネットワークを識別するための部分
 どのネットワークで使われているのかがわかる
ホスト部
 ネットワーク内のコンピュータを識別するための部分
 ネットワーク内のどのコンピュータに割り当てられているかがわかる
クラス
 AからEの5つあり、それぞれネットワークアドレスとホストアドレスがどこで分かれるかが決められている
プライベートIPアドレス
 社内LANなど小規模なネットワークで使用されるIPアドレス
グローバルIPアドレス
 インターネットを利用するときに割り振られるIPアドレス
 「ICANN」→「APNIC」→「プロバイダ」→「ユーザー」
ネットワークアドレス
 ホスト部の2進数のビットが全て0のアドレス
 ネットワーク自体を指す
ブロードキャストアドレス
 ホスト部の2進数のビットが全て1のアドレス
 あるセグメントに接続されている全てのホストにパケットを送信するためのアドレス
ループバックアドレス
 127.0.0.1
 自分自身を示す仮想的なIPアドレス
サブネット
 クラスA.B.Cのネットワークを複数の小さなネットワークに分割したネットワーク
サブネットマスク
 ネットワークアドレスとホストアドレスを識別するための数値
プレフィックス表記
 サブネットマスクの短縮表記、「192.168.0.xxx/yy」の「/yy」のような表記方法
 「255.255.255.0」→「/24」
デフォルトゲートウェイ
 所属するネットワーク内のコンピュータから外部のネットワーク上にあるコンピュータにアクセスする際に、
 出入り口の役割を果たす機器
 ネットワーク内に存在する全ての機器のIPアドレスと、他のデフォルトゲートウェイへの通信経路が設定されている
IPv6
 128ビットアドレス→事実上無限にIPアドレスをデバイスに割り当てられる
 アドレス不足によるNATやPATの実装も不要、IPアドレス自動設定
サブネットプレフィックス
 /〇〇、
インターフェイスID
 リンク上でホストを識別するために使用
 64ビット、手動自動どちらでも設定できる
ルータ
 ネットワーク（LANとLAN、LANとWAN）間の接続を行うネットワークデバイスの一つ
 ネットワーク間のデータ（パケット）配送を行う
ルーティングテーブル
 ルータ内で経路情報を指し示す経路表
 主に「宛先ルート」「パケット転送先」「出力インターフェース」が記載される
ネクストホップ
 パケットが通過できる最も近い次のルータを指す
ARPテーブル
 イーサネット通信のために用いられるIPアドレスとMACアドレスの対照表
ARP
 Address Resolution Protocol
 IPアドレスからEthernetのMACアドレスの情報を得られるプロトコル
コアルータ
 基幹ネットワークのバックボーンを構成するルータで、高速なパケット転送を実現する
 通信キャリアやISPネットワークの通信を中継する
センタールータ
 企業の拠点間WANネットワークで本社拠点・データセンタ等、企業で中心的に利用されるネットワークを接続するために利用される
エッジルータ
 企業の拠点間WANネットワークで支社や営業所等の拠点ネットワークを接続するために利用される
ブロードバンドルータ
 家庭環境やSOHO環境でインターネット回線に接続する目的で利用される
ルーティング
 送信元から宛先にパケットを送信するとき、最適なルートを経由して転送されるようにすること
直接接続
 ルータが直接接続しているネットワークのルート情報
スタティックルーティング
 スタティックルートでリモートネットワークのルート情報を登録すること
ダイナミックルーティング
 ルーティングプロトコルでリモートネットワークのルート情報を登録すること
ディスタンスベクタ型
 ディスタンス（距離）とベクタ（方向）により最適なルートを決定するアルゴリズム
リンクステート型
 どのルータとどのルータが隣接しているかという接続情報（リンクステート）を交換し合い、この情報に基づいて経路を選択する方式
ハイブリッド型
 基本的な仕組みはディスタンスベクタ型（拡張ディスタンスベクタ型とも呼ばれる）
 失った経路情報は、隣のルータから、そのルータが知らなかったら、さらにその隣のルータからと伝言リレー式

■ トランスポート層
 通信の信頼性を確立する層
コネクション型通信
 通信を確立するもの同士で連絡を取り合って制御を行う方式（例：電話）
 代表的なプロトコル「TCP」
ACK
 acknowledgement、確認応答
 データ送受信の確認を意味する信号
コネクションレス型通信
 連絡を取り合って制御はせずに、相手が受け取る準備ができているかにかかわらず、送信する方式（例：メール）
 代表的なプロトコル「UDP」
TCP
 インターネットにおいて標準的に利用されているプロトコル
 信頼性の高い通信を実現
スリーウェイハンドシェイク
 IPネットワーク上でTCPによる接続（コネクション）を確立するための手順
 パケットの送受信を3回行う
Telnet
 ネットワークに接続された機器を遠隔操作するために使用するアプリケーション層プロトコル
 PCにはtelnetクライアント、ルータなどの機器にはtelnetサーバのサービスが有効であることが前提
HTTP
 HyperText Transfer Protocol
 WebサーバとクライアントのWebブラウザがデータを送受信するために使用するアプリケーション層のプロトコル
FTP
 File Transfer Protocol
 特定のコンピュータ間でファイル転送するときに使用するアプリケーション層のプロトコル
SMTP
 Simple Mail Transfer Protocol、TCP上で動作、ポート番号25
 電子メールを送信するために使用するアプリケーション層のプロトコル
TCPヘッダ
 送信元ポート(16bit)、宛先ポート(16bit)、シーケンス番号(32bit)、確認応答番号(32bit)、データオフセット(4bit)、予約(6bit)、フラグ(6bit)、
 ウィンドウサイズ(16bit)、チェックサム(16bit)、緊急ポインタ(16bit)、オプション(可変長)、パディング(可変長※０)
UDP
 User Datagram Protocol、IPの上位のプロトコル
 TCPと比較して信頼性は高くないが、速さやリアルタイム性を求める通信に使用される
RTP
 Real-time Transport Protocol
 音声や映像をストリーミング再生するためのUDP上で動作するプロトコル
RTCP
 Real-time Transport Control Protocol
 RTPの送受信を制御するためのプロトコル
 送出パケット数、受信パケット数、遅延時間、パケット損失率等の通信状況を伝えている
DHCP
 Dynamic Host Configuration Protocol
 LAN内のパソコンやスマートフォンなどにIPアドレス等の情報を自動で割り振り（設定）するためのプロトコル
TFTP
 Trivial File Transfer Protocol
 特定のコンピュータ間でファイル転送するときに使用するアプリケーション層プロトコル
 FTPと違い、ユーザ名とパスワード認証が行われず、転送効率を重視したUDP上で動作する
RIP
 Routing Information Protocol
 ネットワークで接続したルータ間で使用するディスタンスベクタ型ルーティングプロトコル
 RIPv1、RIPv2
UDPヘッダ
 16bit × 4 = 64bit (8バイト)
 送信元ポート(0)、宛先ポート、データ長、チェックサム(0)
